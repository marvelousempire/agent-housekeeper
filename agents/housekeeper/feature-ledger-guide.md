# Feature Ledger Guide

## Purpose

The Feature Ledger tracks buildable work as small Markdown tickets.

## Structure

```text
features/
├── README.md
├── ledger.md
└── tickets/
```

## Ticket Rule

One feature equals one Markdown ticket.

## Ticket Fields

Each ticket should include:

- Feature ID
- Feature name
- Status
- Valid intent
- User value
- Description
- Acceptance criteria
- Validation method
- Related files

## ID Format

```text
FEAT-0001-feature-name.md
```
