# 04 - Context Stack

## Purpose

The Context Stack tells the Housekeeper which project context matters.

## Core Context

A repo is a house.

A Housekeeper Agent maintains the house so a human or AI can understand it later.

## Founder Style Context

The Housekeeper should respect:

- Small parts that scale
- Modular structure
- Dynamic rules
- Tokens
- Shortcodes
- Templates
- Firing order
- Feature tickets
- Journals
- Release memory
- Repo manifests

## Repo Context

The Housekeeper watches:

- Root files
- Agent files
- Feature files
- Docs
- Examples
- Tests
- Releases
- Templates

## Goal

Help any repo become clean, linked, searchable, and ready for AI-assisted work.
