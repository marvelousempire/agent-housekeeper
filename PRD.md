# PRD - Agent Housekeeper

## Product Name

Agent Housekeeper

## Product Type

Reusable repo maintenance standard for AI-assisted projects.

## Problem

A repo can become hard to manage when notes, features, decisions, prompts, docs, and releases are scattered.

An AI helper also needs a clear starting point, a reading order, and a set of rules for how to keep the repo organized.

## Goal

Create a reusable structure that teaches an AI helper how to maintain a repo.

## Users

- Founders
- Builders
- Repo maintainers
- Product planners
- AI-assisted development teams

## Core Features

## 1. System Prompt Folder

A primary folder tells the helper where to start.

## 2. Firing Order

A numbered order tells the helper which rules to read first.

## 3. Repo Manifest

A manifest lists the important files and folders.

## 4. Feature Ledger

A ledger tracks buildable work as small Markdown tickets.

## 5. Journal System

Journal files preserve project memory and decisions.

## 6. Validation System

Validation files help check what is complete, what needs review, and what is missing.

## 7. Repo Transformation Guide

A guide explains how to reshape another repo into the Housekeeper pattern.

## Acceptance Criteria

- System prompt folder exists.
- Firing order exists in Markdown and JSON.
- Persona file exists.
- Operating manual exists.
- Knowledge source file exists.
- Feature ledger exists.
- Templates exist.
- README explains how to start.

## Success Definition

A new project can point to this repo and understand how to set up a clean, documented, feature-tracked, validation-ready repo structure.
