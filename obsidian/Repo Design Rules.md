# Repo Design Rules

## Purpose

This note summarizes the repo design rules in wiki form.

## Required Root Files

- README.md
- ABOUT.md
- PRD.md
- LICENSE
- repo-manifest.md
- release-ledger.md

## Required Folders

- agents/
- docs/
- examples/
- features/
- obsidian/
- obsidian/journal/
- tests/

## Main Rule

Every important repo cell needs a place.

## Micro-Document Rule

Prefer small focused Markdown files.

## Ledger Rule

Buildable work becomes a feature ticket.

## Journal Rule

Important decisions become journal notes.

## Manifest Rule

Major structure changes update the repo manifest.

## Release Rule

Major states update the release ledger.
